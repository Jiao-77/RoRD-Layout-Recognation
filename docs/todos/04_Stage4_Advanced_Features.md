  - [ ] 设置 Optuna 数据库（SQLite 或 PostgreSQL）

- [ ] 定义搜索空间

  - [ ] 创建 `tools/hyperparameter_tuning.py`

**验收标准**:
- [ ] Optuna 搜索可正常运行

## ⚠️ 风险与注意事项

1. **实验管理**
   - 数据隐私：敏感数据不上传云端
   - 成本管理：W&B 免费额度有限
   - 网络依赖：离线环境需配置本地 MLflow

2. **超参优化**
   - 搜索时间长：可能需要数天或数周
   - GPU 资源消耗：建议分布式搜索
   - 过拟合风险：避免过度优化验证集

3. **性能优化**
   - 精度损失：量化和蒸馏可能降低精度
   - 兼容性问题：不同 GPU 推理性能差异大
   - 维护成本：多个模型版本增加维护负担
