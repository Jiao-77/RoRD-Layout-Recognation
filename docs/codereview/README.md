## 问题分类

审查报告中的问题按以下严重程度分类：

| 标记 | 严重程度 | 说明 |
|------|---------|------|
| 🔴 | 严重 | 可能导致程序崩溃或数据损坏 |
| 🟠 | 中等 | 影响功能正确性或可维护性 |
| 🟡 | 轻微 | 代码风格、文档等改进建议 |
| 🔵 | 性能 | 性能优化建议 |
| 🟣 | 架构 | 设计和架构改进建议 |
| 🔒 | 安全 | 安全漏洞或风险 |

## 修复流程

1. 按优先级排序问题（严重 > 中等 > 轻微）
2. 为每个问题创建修复分支
3. 修复后更新报告中的状态
4. 提交 PR 并关联问题编号
